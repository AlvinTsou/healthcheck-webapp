🔒 安全性與隱私權宣告

網頁安全性與隱私權宣告

本系統針對您上傳的健康報告與圖檔，在隱私處理安全性上達到企業級的防護標準，具體安全防護機制可總結為以下四點：

1. 零本地殘留與暫存（Zero Local Retention）
  • 無硬碟寫入：所有上傳的健檢報告與圖檔僅在應用程式伺服器的記憶體 (RAM) 中進行即時流式讀取。
  • 即時垃圾回收：API 處理完畢並回傳分析結果後，記憶體中的檔案資料立即釋放。伺服器本地磁碟均完全不留存任何客戶檔案，從根本上杜絕了伺服器遭入侵時資料被竊取的風險。
2. 傳輸層全程加密（End-to-End Encryption in Transit）
  • 用戶端至後端：瀏覽器與 WebApp 後端之間的檔案傳輸，強制採用 HTTPS (TLS 1.2 / TLS 1.3) 進行高強度加密。
  • 後端至雲端：WebApp 與 Google Cloud Platform (GCP) 之間的 SDK 呼叫，均透過 Google 的安全通道進行加密傳輸，防止任何中間人攻擊 (MITM) 或資料竊聽。
3. 去識別化日誌設計（PII-Free Logging）
  • 無個人敏感資訊：系統運行日誌定位為使用量與除錯用途，絕不抓取或記錄客戶報告中的任何生理數值、診斷結果或個人身份訊息（如姓名、身分證字號等 PII 資訊）。
  • 記錄範圍：僅保留時間戳記、去識別化邀請碼、原始檔名與檔案大小作為配額統計。
4. 企業級 Google Cloud 隱私承諾與合規性
  • 無模型訓練風險：系統呼叫之 Vertex AI Gemini API 隸屬於 Google Cloud 企業級服務。根據 GCP 服務條款，客戶的輸入資料與分析內容，絕不會被 Google 用於訓練或微調其公共基礎模型。
  • 雲端安全合規：資料在 Google Cloud 基礎設施中處理，受控於 GCP 的安全防護機制，符合多項國際安全認證（如 ISO 27001、SOC 2/3、及符合醫療隱私的 HIPAA 架構）。
